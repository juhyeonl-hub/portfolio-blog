## Background

Using AI as an auxiliary tool vs designing the development loop itself around AI.

![Traditional vs AI-first workflow](https://raw.githubusercontent.com/juhyeonl-hub/portfolio-blog/main/journal/images/09_comparison.png)

## Analysis

Two things are required:

**First, clear instruction ability.** Precise specifications become the development skill itself.

**Second, verification ability.** Judge whether AI's code is correct.

![AI code verification checklist](https://raw.githubusercontent.com/juhyeonl-hub/portfolio-blog/main/journal/images/09_checklist.png)

![AI-first development loop](https://raw.githubusercontent.com/juhyeonl-hub/portfolio-blog/main/journal/images/09_ai_loop.png)

## Reflection

AI-first development demands more from the developer, not less. You need the entire design in your head, judge output quality, and pinpoint exactly what's wrong. A developer's value in the AI era is distinguishing good code from bad — and that ability comes from fundamentals.
